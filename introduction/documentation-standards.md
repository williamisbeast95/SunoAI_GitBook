# Documentation Standards

These standards ensure consistency across the GitBook.

## Writing Style

- Clear and direct language
- Avoid unnecessary complexity
- Prefer practical examples over theory

## Structure

Each page should include:

- Overview
- Key concepts
- Practical usage
- Tips or best practices

## Version Awareness

When features differ between model versions, label them clearly.

## Consistency Rules

- Use consistent headings
- Keep formatting uniform
- Avoid duplicate content

## Updates

This documentation is intended to evolve. Update pages when new features or behaviors are confirmed.
